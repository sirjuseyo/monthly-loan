## 0. 한 줄 요약

써주세요 웹 진입 시 **자동 노출되는 강제 고지형 팝업**. 포인트 활용 대출 구조를 설명하고, 다음 주 지자체 현장 실사 + 민원에 대비해 정책·절차·이용방법을 **처음부터 고지**한다. 끝까지 스크롤 + 체크박스 3개를 모두 체크해야만 닫힌다.

## 1. 목적·배경

- 써주세요연계대부(ALBITF)는 등록 대부업체. 공무원의 요청 사항은 **"사용자에게 처음부터 정책·절차·이용방법을 노티하라"**는 것 → 본 팝업이 그 고지물.
- 핵심 메시지: **"원금의 일부와 이자 전부를 포인트로 상환하는 포인트 활용 대출"**.
- 법적 고지(금리·기간·부대비용·경고 등)는 **웹 상·하단에 이미 존재** → **팝업에는 중복 포함하지 않는다.**
- **웹 적용 대상 = 2개 페이지:** `https://monthly-loan.sirjuseyo.com/` · `https://www.sirjuseyo.com/`

## 3. 필수 구현 요건 (★ 반드시 지킬 것)

### 3.1 강제 고지 동작 (이미 소스에 구현됨)
- **모든 콘텐츠 즉시 전체 노출.** 토글/아코디언(클릭 펼침) **금지**.
- **상단 X 없음 / 배경 클릭 닫기 없음 / ESC 닫기 없음.**
- 닫는 유일한 경로 = **본문 최하단까지 스크롤 + 체크박스 3개 전부 체크** → `확인하고 시작하기` 버튼 활성화 → 클릭.
- 상단 진행바 + 하단 상태 텍스트("끝까지 스크롤해서 읽어주세요" / "확인 항목 N/3 — 남은 항목을 체크해주세요" / "확인 완료 — 닫을 수 있어요")로 안내.

### 3.2 ★ 미구현 → 쮸티가 반드시 추가: 재노출 방지(저장)
- 현재 소스의 `close()`는 모달만 닫는다. **'오늘 하루 보지 않기' / 재노출 방지 로직이 없다.** (아티팩트 환경 storage 미지원 때문)
- **쮸티 작업:** `close()` 시점에 **cookie 또는 localStorage에 오늘 날짜를 저장**하고, **페이지 진입 시 저장된 날짜 === 오늘이면 팝업을 띄우지 않도록** 분기 추가. (소스 내 해당 위치에 주석 `⚠️ 쮸티:` 표기됨)
- 정책(하루 1회 / 매 진입 / N일) 은 대표 확인 후 결정. 기본 권장: **하루 1회.**

### 3.3 ★ 가로 폭·통합
- **검증 사실:** 웹 컨테이너는 `.wrap { max-width: 480px; margin: 0 auto }`, `body { padding: 0 }` (저장소 `github.com/sirjuseyo/monthly-loan` 의 `index.html`, `2026-06/index.html` 등에서 확인).
- 팝업 카드 폭은 **480px (`.popup { max-width:480px }`)** 로 웹 컨테이너와 **동일**하게 맞춰져 있음. 오버레이 좌우 패딩 0.
- **권장 통합:** 팝업을 `.wrap` 내부에 주입하거나, 사이트 컨테이너 폭이 바뀌어도 일치하도록 `.popup` max-width를 사이트와 같은 값으로 유지. (정확한 주입 파일·시점은 대표/쮸티 결정 사항.)

### 3.4 폰트
- 소스는 **Pretendard**를 jsDelivr CDN으로 import, 폴백 `-apple-system, 'Apple SD Gothic Neo', 'Malgun Gothic', sans-serif`.
- 사이트가 이미 폰트를 로드 중이면 중복 로드 정리 또는 동일 폰트로 통일. 오프라인/자체호스팅 필요 시 Pretendard 번들.

### 3.5 법적 고지 (팝업 미포함 — 그대로 둘 것)
- 금리(연 12% 등)·대출기간·부대비용·경고 문구 등 **법적 고지는 팝업에 넣지 않는다.** 웹 상·하단 고지가 정본. (중복 금지)

- 써주세요.는
- 1. 「포인트를 활용하는 대안 신용 대출」 플랫폼입니다.
- 2. 원금의 일부와 이자 전부를 포인트로 상환합니다.

**3. 🔒 포인트 활용, 이렇게 운영됩니다**
1. 써주세요.는 신청 시 포인트를 선적립하는 포인트 활용 대출입니다. 승인 전까지 고객님의 마이 포인트에 그대로 보관됩니다.
2. 그 선적립 포인트로 원리금을 상환합니다. 대부업법상 원리금을 포인트로 상환하는 것은 적법합니다. 승인 후에는 원금의 일부와 이자를 현금 대신 이 선적립 포인트로 상환하며, 상환 시점에 차감되어 그만큼 현금 부담이 줄어듭니다.
3. 크레딧 미션 포인트는 신용조회비용으로 사용되며, 승인 시 차감됩니다.
4. 선적립 포인트는 승인 및 상환 시에만 차감되고, 한도가 발생하지 않으면(불승인) 전액 환전해 드립니다. 가져가는 돈이 아니라 맡겨두는 예치금입니다.

- ③ 이자 미션 — 이자 상환에 현금 대신 · 상환 시 차감
- 활용 포인트는 상환 전까지 마이 포인트에 그대로 보관됩니다.
- 단, 크레딧 미션 포인트는 승인 시 차감됩니다.
- ☑ 체크 1: 포인트 활용 대출의 구조를 확인했습니다

**6. 💳 크레딧 미션**
- 크레딧 미션 포인트는 신용조회비용으로 사용됩니다.

- 활용 포인트 중 일정 포인트가 해당 비용으로 사용됩니다.
- 마이 포인트에 고스란히 보관되어 있다가 승인 후 차감됩니다.
- 한도가 발생하지 않으면(불승인) 전액 환전 처리됩니다.
- 신용조회비용의 시가는 10,000원이지만, 고객님의 주머니 사정을 감안하여 당사가 일정 부분 부담해 할인해 드립니다.
- 활용 기준: 10만 4~5% / 20만 2.5~3% / 30만 2~3% / 50만 2~3%
- 신청 금액에 따라 상이하며, 웨이팅 대출은 신청 시기에 따라 추가 할인 혜택이 있습니다.

## 8. 앱(Android·iOS) 적용 — 별도 작업 (앱 쮸티 대상)

본 v5는 **웹 전용**(위 2개 페이지). 앱에는 이 HTML/CSS/JS를 그대로 올릴 수 없으므로 별도 작업이 필요하다.

- **방식:** (B) 각각 **Kotlin(Android) & Swift(iOS) 네이티브로 재구현.** 콘텐츠·강제 고지 동작(전량 노출 / 상단 X·배경·ESC 닫기 없음 / 최하단 스크롤 + 체크 3개)은 **동일**하게, UI만 네이티브로. — (나)앱용 = (가)웹용 + 10%.
  - [참고] (A) 웹뷰로 이 HTML을 그대로 띄우는 빠른 방법도 있으나, 방향은 **네이티브 재구현**.
- **★ 가로 폭 — 앱 쮸티 확인 필요(질문 항목):** 웹은 컨테이너 `.wrap` = **480px 고정**. 앱은 보통 **기기 화면 폭(가변, 360~430dp 등)**이라 웹의 480px 고정과 다를 가능성이 크다. 써주세요 앱의 실제 레이아웃 폭을 **확인 후 대응**할 것(480px 고정 vs 기기 폭 100% 중 결정).
- **재노출 방지:** 웹 = cookie/localStorage(오늘 날짜) / 앱 = **SharedPreferences(Android)·UserDefaults(iOS)** 로 동일 정책 구현.
- **법적 고지:** 앱에 이미 있는 고지와 중복하지 말 것(웹과 동일 원칙).

  const overlay=document.getElementById('overlay'),body=document.getElementById('body'),bar=document.getElementById('bar'),
        cta=document.getElementById('cta'),status=document.getElementById('status'),statusTxt=document.getElementById('statusTxt'),
        reopen=document.getElementById('reopen'),cbs=[...document.querySelectorAll('.cb')];
  let scrolledBottom=false;
  function refresh(){
    const max=body.scrollHeight-body.clientHeight;
    bar.style.width=(max<=0?1:Math.min(1,body.scrollTop/max))*100+'%';
    if(max<=0||body.scrollTop+body.clientHeight>=body.scrollHeight-8) scrolledBottom=true;
    const checked=cbs.filter(c=>c.checked).length, ready=scrolledBottom&&checked===cbs.length;
    cta.classList.toggle('on',ready); cta.disabled=!ready;
    if(ready){status.classList.add('ready');statusTxt.textContent='확인 완료 — 닫을 수 있어요';}
    else{status.classList.remove('ready');
      statusTxt.textContent=!scrolledBottom?'끝까지 스크롤해서 읽어주세요':`확인 항목 ${checked}/${cbs.length} — 남은 항목을 체크해주세요`;}
  }
  function close(){ if(cta.disabled)return; overlay.hidden=true; reopen.hidden=false;
    /* ⚠️ 쮸티: cookie/localStorage(오늘 날짜) 저장 → 재노출 방지 */ }
  body.addEventListener('scroll',refresh,{passive:true});
  cbs.forEach(c=>c.addEventListener('change',refresh));
  cta.addEventListener('click',close);
  reopen.addEventListener('click',()=>{overlay.hidden=false;reopen.hidden=true;body.scrollTop=0;scrolledBottom=false;cbs.forEach(c=>c.checked=false);refresh();});

</script>
